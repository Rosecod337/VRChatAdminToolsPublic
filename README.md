# VRChat Admin Tools

Открытый исходный код двух Windows-приложений для администраторов VRChat-сообществ:

- **VRChat Log Analyzer** читает локальные `output_log_*.txt`, показывает события текущей сессии, заметки, историю, диагностику и результаты Crash Risk Analyzer.
- **VRChat Key Admin** управляет лицензиями собственного сервера.

Репозиторий также содержит нейтральный `server-template` для самостоятельного размещения. Он создаёт новую пустую базу PostgreSQL и не содержит production-конфигурацию, ключи, базу данных или закрытые сервисы проекта.

## Готовая версия

Последний официальный установщик: [`v1.0.3`](../../releases/latest).

Исходный код в репозитории предназначен для самостоятельной сборки и собственного сервера. В нём по умолчанию отключены официальные автоматические обновления, а API настроен на `http://localhost:8080`. Для готового приложения используйте установщик из Releases.

## Быстрый запуск

Требования: Windows 10/11, Node.js 20 или новее, npm и PostgreSQL.

```powershell
npm ci
Copy-Item .env.example .env
npm run server:dev
npm run admin:dev
npm run client:dev
```

Перед запуском сервера укажите в `.env` собственный `DATABASE_URL`, а также разные случайные значения для `LICENSE_PEPPER` и `ADMIN_TOKEN`. Не используйте значения из примера в рабочей среде.

Адрес self-hosted API задаётся в `apps/client/config.json`. Адрес сервера и `ADMIN_TOKEN` для VRChat Key Admin вводятся в интерфейсе самого приложения.

Сборка установщиков:

```powershell
npm run build:client
npm run build:admin
```

Пошаговая инструкция для PostgreSQL и Railway находится в [docs/RAILWAY.md](docs/RAILWAY.md).

## Структура

- `apps/client/` - пользовательское Electron-приложение.
- `apps/admin/` - Electron-приложение управления ключами.
- `server-template/` - совместимый Express/PostgreSQL backend для собственного сервера.
- `packages/parser/` - парсер локальных логов VRChat.
- `tests/` - автоматические проверки.
- `runtime-config.json` - подписанная конфигурация подключения официального установщика; self-hosted сборка её не использует.

## Безопасность и ограничения

Приложение не внедряется в VRChat, не читает память игры и не изменяет клиент. Оно работает только с локальными логами и разрешёнными пользователем API-запросами. Логи могут не содержать точный состав инстанса, владельца портала или доказательство виновника сбоя.

Не добавляйте в Git `.env`, cookie, лицензионные ключи, токены, дампы PostgreSQL, `node_modules`, `.build`, `release` и собранные установщики. Cookie VRChat вводится пользователем вручную, хранится локально и не должен отправляться на self-hosted сервер.
